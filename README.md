# GateLog - Smart Parcel Management System

---

## 🧠 Introduction

Every college campus deals with the same issue — parcels piling up at gates, missing deliveries, no proper verification, and zero accountability.

GateLog is a structured solution built to address this problem using a simple, role-based system.

Instead of focusing on heavy frameworks, this project emphasizes:
- Clear system design  
- Real-world usability  
- Strong fundamentals  

---

## 🎯 Problem Statement

Campus parcel handling systems lack:
- Centralized tracking  
- Identity verification during pickup  
- Transparency between students and security  

This leads to:
- Lost or misplaced parcels  
- Unauthorized collection  
- Poor communication  

GateLog solves this by introducing a digital workflow from parcel arrival to verified delivery.

---

## ⚡ Solution Overview

GateLog provides a role-based parcel management system with:

- Controlled user access  
- Structured parcel logging  
- Verified pickup process  
- Transparent tracking  

---

## 🔥 Features

### 🔐 Role-Based System
- Separate dashboards for:
  - Admin  
  - Security Guards  
  - Students  

### ✅ Admin Approval Workflow
- New users require admin approval before access  
- Ensures system security  

### 📦 Parcel Management
- Security logs incoming parcels  
- Assigns them to students  
- Tracks status (Pending / Accepted / Rejected)  

### 🔍 Verified Pickup
- Students confirm identity before collection  
- Prevents misuse  

### 📊 Student Dashboard
- View pending parcels  
- Track parcel history  

### 🧭 Lost & Found Board
- Report and recover lost items

## 👥 User Roles

### 🛠️ Admin
- Approves or rejects user registrations  

### 🛡️ Security Guard
- Logs parcels  
- Verifies pickup  
- Manages lost & found  

### 🎓 Student
- Views assigned parcels  
- Accepts/rejects deliveries  
- Tracks parcel history
